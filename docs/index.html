local-gpu-scheduler

Like Slurm's srun, but for your local machine. Set up in 60 seconds.

curl -sSL https://raw.githubusercontent.com/GindaChen/local-gpu-scheduler/main/install.sh | bash

Quick Start

use srun to run your job
# Run a single GPU job
$ srun python train.py
srun: requesting 1 GPU(s) from scheduler (pid=42053)...

# Multi-GPU
$ srun -n 4 torchrun --nproc_per_node=4 train.py
srun: acquired GPU(s) 0,1,2,3 (job=b7d2e9f1)

# Check status
$ srun jobs
a3f1c8e2 done 0 python train.py
b7d2e9f1 running 0,1 torchrun train.py
start the global scheduler server (one time)
# Start server + TUI in tmux (default)
$ srun-server start
✓ Scheduler started in tmux session 'gpusched'

# Or foreground only
$ srun-server start --no-tmux
gpusched server on :9123

Commands

srun

Run a GPU job

srun python train.py
srun -n 4 torchrun train.py

srun-server

Manage the scheduler

srun-server start
srun-server stop

srun-agent

Auto-retry with GPU fix

srun-agent python train.py

srun status / jobs

Query the scheduler

srun status
srun jobs

srun update

Self-update

srun update

How it works

1

srun sends your PID to the server

Requests N GPUs. Blocks until they're available (FIFO).

2

Server checks nvidia-smi

Polls every second for free GPUs.

3

GPUs assigned → your command runs

CUDA_VISIBLE_DEVICES is set, then exec replaces srun with your command.

4

Process exits → GPUs released

Server monitors your PID. When done, GPUs go back to the pool.
